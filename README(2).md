## 功能特点

### 1. 🌤️ 天气穿搭推荐
根据温度、湿度、紫外线指数、天气状况，智能推荐合适季节和厚度的穿搭。

### 2. 👕 已有衣物搭配推荐
用户搜索并输入自己的已有衣物，系统分析品类覆盖情况，智能推荐补全方案。

### 3. 🎨 风格偏好选择
支持 40+ 种纯风格标签：休闲、简约、复古、潮流、时尚、优雅、学院、甜美、酷感、通勤、商务、户外、运动、正装、街头、美式、英伦、法式、意式、德式、韩系、日系、中式、国潮、北欧、牛仔、皮革、针织、工装、军旅等。

### 4. ⭐ 推荐结果评分
用户对每次推荐结果进行 0-5 星评分，评分数据持久化保存，支持统计分析。

### 5. 🌐 中英文切换
完整支持中英文双语界面。

### 6. 🔍 衣物关键词搜索
中英文混合关键词搜索，实时展示匹配结果。

## 机器学习算法

本系统同时实现了**两套推荐方案**：

### 方案一：基于规则的推荐系统（当前前端使用）
| 方法 | 用途 |
|------|------|
| 关键词字典匹配 | 从衣物描述文本提取风格标签 |
| 阈值映射 | 温度 → 季节（夏季/春秋/冬季） |
| 多维度筛选 + 排序 | 按性别 → 风格 → 季节差 → 厚度差 推荐 |
| 品类覆盖率检查 | 分析已有衣物的品类空缺，推荐补全 |
| 正则表达式文本预处理 | 衣物搜索 + 术语翻译 |

### 方案二：基于机器学习的推荐（model_training.py，已训练好模型）
| 算法 | 用途 | 原理 |
|------|------|------|
| **余弦相似度** (cosine_similarity) | 物品相似性计算 | 基于物品特征向量计算相似度 |
| **基于物品的协同过滤** (Item-based CF) | 个性化推荐 | 加权求和预测用户评分 |
| **LabelEncoder** | 特征工程 | 文本类别转数值 |
| **随机森林分类器** (RandomForestClassifier) | 风格分类 | 多棵决策树投票 |
| **梯度提升回归** (GradientBoostingRegressor) | 天气适配度评分 | 梯度提升树 |
| **用户-物品矩阵** | 用户偏好分析 | 稀疏矩阵构建用户相似度 |

## 安装与运行

### 环境要求
- Python 3.8+
- pandas, numpy, scikit-learn, streamlit, openpyxl

### 步骤一：安装依赖

```bash
pip install -r requirements.txt
```

### 步骤二：训练模型（可选 — 模型已预训练好）

如果想重新训练模型：

```bash
python train.py
```

训练过程会：
1. 加载 xlsx 数据
2. 训练物品推荐模型（随机森林）
3. 训练穿搭推荐模型（梯度提升）
4. 构建物品相似度矩阵（余弦相似度）
5. 保存所有模型文件到 `model/` 目录

### 步骤三：验证系统功能

```bash
python verify_system.py
```

### 步骤四：启动 Web 界面

```bash
streamlit run streamlit_app.py
```

浏览器会自动打开 `http://localhost:8501`

### 步骤五：查看实验 Notebook

```bash
jupyter notebook backend_core.ipynb
```

## 数据说明

### 数据格式
项目使用 **Excel (.xlsx)** 格式，按性别分三个子目录：

| 目录 | 数据量 | 说明 |
|------|--------|------|
| `data/male/` | 男装 | 含 `label_male.xlsx`（单品标签）+ `look_male.xlsx`（搭配） |
| `data/female/` | 女装 | 含 `label_female.xlsx`（单品标签）+ `look_female.xlsx`（搭配） |
| `data/child/` | 童装 | 含 `label_child.xlsx`（单品标签）+ `look_child.xlsx`（搭配） |

### 单品标签字段
| 字段 | 类型 | 说明 |
|------|------|------|
| itemID | 字符串 | 衣物唯一标识 |
| category | 字符串 | 品类（inner_top / mid_layer_top / outerwear / bottom / shoes / accessory / onepiece） |
| title | 文本 | 衣物描述（英文字段，如 "minimalist solid versatile top"） |
| style | 文本 | 风格描述（英文字段，如 "vintage casual"） |
| season | 整数 | 适用季节（0=夏季, 1=春秋, 2=冬季） |
| thick | 整数 | 厚度（0=薄, 1=中等, 2=厚） |
| gender | 字符串 | 性别（男 / 女 / 童） |

### 天气字段
| 字段 | 类型 | 说明 |
|------|------|------|
| temperature | 浮点数 | 温度（℃） |
| humidity | 浮点数 | 湿度（%） |
| uv_index | 整数 | 紫外线指数 |
| weather_condition | 字符串 | 天气状况（sunny / cloudy / windy / rainy / snowy 等） |

| `encoded_looks.csv` | CSV | 编码后的穿搭特征矩阵 |
| `items_data.csv` | CSV | 物品原始数据副本 |
| `looks_data.csv` | CSV | 穿搭原始数据副本 |
| `weather_data.csv` | CSV | 天气参考数据 |
| `rating_data.csv` | CSV | 用户评分参考数据 |

如果想重新训练模型：

## 推荐流程

### 天气推荐流程
```
用户输入 (温度, 湿度, UV, 天气, 性别, 风格)
      │
      ▼
┌──────────────────────────┐
│ 天气分析                    │
│ - heat_protection (T≥30)  │
│ - warm_protection (T≤5)   │
│ - rain_protection         │
│ - wind_protection         │
│ - uv_protection           │
└──────────────────────────┘
      │
      ▼
┌──────────────────────────┐
│ 目标季节 / 厚度映射          │
│ T≥25 → 夏季 → 薄款           │
│ T≤10 → 冬季 → 厚款           │
│ 其他 → 春秋 → 中等厚度        │
└──────────────────────────┘
      │
      ▼
┌──────────────────────────┐
│ 衣物筛选 + 排序              │
│ 1) 按性别筛选                 │
│ 2) 按风格关键词匹配           │
│ 3) 按 |season - target| +    │
│    |thick - target| 升序      │
│ 4) 取 Top 1                  │
└──────────────────────────┘
      │
      ▼
 输出推荐穿搭 + 小贴士
```

### 已有衣物推荐流程
```
用户搜索已有衣物 → 加入 owned_items
      │
      ▼
┌──────────────────────────┐
│ 品类分析                    │
│ 已有: [内搭, 下装]           │
│ 缺失: [外套, 鞋履, 配饰]     │
└──────────────────────────┘
      │
      ▼
┌──────────────────────────┐
│ 缺失品类推荐                │
│ 按季节+厚度+风格筛选          │
└──────────────────────────┘
      │
      ▼
 输出推荐搭配 + 评分入口
```

## 特色亮点

1. **两套推荐方案** — 规则系统（可解释性强）+ ML 系统（协同过滤+集成学习）
2. **多语言支持** — 中英文双语界面，英文数据自动翻译
3. **用户反馈闭环** — 推荐结果评分，数据持久化，支持持续改进
4. **实时搜索** — 关键词模糊匹配，支持中英文混合搜索
5. **完整的工程化** — 数据加载 → 模型训练 → 系统验证 → 界面展示 全流程
